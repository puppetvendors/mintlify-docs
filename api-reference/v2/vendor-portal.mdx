## Overview

The Vendor Portal API provides a dedicated set of endpoints (`/v2/vendor/*`) for vendor self-service operations. These endpoints require a **vendor-scoped JWT token** obtained via `POST /v2/authenticate` with vendor credentials.

Vendor tokens are automatically scoped to the authenticated vendor's data — no `vendorId` parameter is needed.

## Authentication

Vendor tokens are obtained by authenticating with vendor credentials:

```bash
curl -X POST "https://api.puppetvendors.com/v2/authenticate" \
  -H "Content-Type: application/json" \
  -d '{"email": "vendor@example.com", "password": "your-password", "shopId": "YOUR_SHOP_ID"}'
```

The returned JWT token includes the vendor scope and should be passed in the `x-access-token` header.

## Scope Restrictions

- Vendor tokens **cannot** access merchant-scoped endpoints (`/v2/vendors`, `/v2/commissions`, etc.)
- Vendor tokens are **auto-scoped** — data is automatically filtered to the authenticated vendor
- Merchant tokens **cannot** access vendor portal endpoints (`/v2/vendor/*`) — they receive 403
